# Iframe Viewer 验收清单

## 范围

这份文档只定义嵌入式 iframe viewer 的验收基线。

纳入范围：

- 原始 `json` / `jsonp` / `jsonl` 页面内的自动接管
- iframe 内的树、右侧工作区、搜索弹层、快捷工具、行内链接和图片预览
- 与 WXT 介入前最后稳定版嵌入式 viewer 在视觉和交互上的一致性

不在范围内：

- detached viewer 独立窗口的 standalone 布局
- 独立的 `viewer.html`
- iframe 之外的 toolkit 页面行为

新增纳入范围：

- `raw` / `pretty` 这类可解析字符串值的“在新窗口按 JSON 打开”流程
- 点击浏览器工具栏图标后的 launcher 入口

## 参考基线

参考行为来自最后稳定的 legacy embedded viewer：

- [viewer.html](/Users/y10n/tmp/crx_ext_JsonMate/public/json-mate/viewer.html)
- [json-mate-viewer.js](/Users/y10n/tmp/crx_ext_JsonMate/public/json-mate/js/json-mate-viewer.js)

新的实现内部可以不同，但验收标准只看用户可见的行为和交互质量，不看内部是否“更现代”。

## 工作规则

- 没有在真实浏览器里明显正确，就不能标记完成。
- 如果某个能力只存在于 inspector、debug DOM 或代码层面，但用户肉眼感知不到，不算完成。
- 不能拿别处的变通入口冒充目标位置已经完成。
- iframe 模式不能串入 standalone viewer 的控件或流程。
- 每个完成项在可行时都应至少配一条自动化验证。

## 验收列表

### A. Iframe 接管壳

- [ ] 打开受支持的原始 payload 页面时，会在当前页面内注入 iframe 接管，而不是新开标签页。
- [ ] iframe 覆盖当前页面的方式与稳定 legacy 版一致。
- [ ] 嵌入壳中不出现 standalone 专属控件。
- [ ] 嵌入壳中不出现 detached viewer 专属控件。
- [ ] 嵌入壳中不出现 pending payload 专属控件。
- [ ] 点击浏览器工具栏图标后，打开的是同一套改好的 iframe viewer 壳，而不是旧的 standalone 坏壳。
- [ ] 如果当前页面选中了可解析的 JSON / JSONP / JSONL，launcher 会直接载入该内容。
- [ ] 如果当前页面选中了文本但当前还不能整体解析，launcher 会用同一套 viewer 壳打开，并把文本带到手工输入区。
- [ ] 如果当前页面没有选中任何文本，launcher 会打开同一套 viewer 壳，并直接提供手工输入区。
- [ ] launcher 空态不能只是空白壳；至少要同时给出手工输入区和可直接体验的 fixture 入口。
- [ ] launcher 空态应允许直接输入一个地址并在新标签页打开。
- [ ] 工具栏 launcher 默认使用新标签页，而不是弹窗。

### B. 树交互

- [ ] 点击任何可见树行，都能稳定选中对应节点。
- [ ] 展开/收起点击区与“选中行”的点击区彼此分离。
- [ ] 数量 badge、图标、值预览不会意外抢走行选择。
- [ ] 首次进入后的第一次点击手感可靠，不会出现明显“点空气”的感觉。
- [ ] 树选择总是同步更新右侧 `Path`、`Key` 和编辑器值。

### C. 右侧工作区

- [ ] 嵌入式工作区的观感和节奏应接近 legacy embedded panel，而不是 standalone viewer。
- [ ] 右上角独立的 logo handle 可以收起面板。
- [ ] 收起时只保留右上角同一个 logo handle，不保留右侧背景。
- [ ] 再次点击这个 logo handle，会把右侧控制完整恢复出来。
- [ ] 面板开合状态对用户来说清晰且稳定。
- [ ] 工作区收起后，点击树节点不会把它自动重新打开。
- [ ] 面板内不包含仅属于 standalone 模式的控件。
- [ ] 设置图标作为全局入口放在合理位置，而不是挂在显示选项头部里。

### D. URL 与媒体入口

- [ ] 像 `homepage` 这种普通 URL 值，在树值旁边就有明显可见的跳转入口。
- [ ] URL 入口应一眼可见，不能只藏在 inspector 侧边。
- [ ] 点击 URL 入口会按预期开新标签页或新窗口。
- [ ] 节点被选中时，URL 小图标也会跟随反色，不会因为颜色不对而看不清。
- [ ] `avatar`、`logo`、`thumbnail` 这类字段继续保持图片预览行为。
- [ ] 非图片 URL 不会被误判成图片。

### E. 搜索弹层

- [ ] 搜索按钮与展开/收起按钮属于同一组树操作按钮。
- [ ] 每次点击搜索按钮都能弹出居中的搜索层。
- [ ] 打开搜索后，焦点落在输入框里。
- [ ] 在 `Keys` / `Values` 之间切换后，焦点仍保持在输入框里。
- [ ] 再次打开搜索时，会保留上一次使用的是 `Keys` 还是 `Values`。
- [ ] 输入搜索词时，不会被重新选中、重置或覆盖。
- [ ] 搜索结果相关性正常，排序对用户可理解。
- [ ] 选择搜索结果后，行为与直接点击树节点一致，并同步右侧工作区。
- [ ] 搜索历史会保留，并且能区分这是 `Key` 搜索还是 `Value` 搜索。

### F. 编辑与工具

- [ ] 选中节点后，右侧编辑器立即出现该节点的当前值。
- [ ] 节点已选中时，右侧 value 编辑器可以持续输入，不会被状态同步立即改回旧值。
- [ ] `Path` 字段不显示 `Root.` 前缀，只显示实际可编辑路径。
- [ ] `Apply edit` 会更新树，并保持选择状态一致。
- [ ] 当当前值是数字/布尔等非字符串类型，而输入文本无法按该类型解析时，不会静默失败。
- [ ] 这类失败会给出明确分支：忽略本次写入，或按字符串保存。
- [ ] `Undo` / `Redo` 的实际行为与稳定 legacy 版一致。
- [ ] `Single line` 在适用时会让结构值明显变成真正的单行展示。
- [ ] 上下文工具只在确实适用于当前值时出现。
- [ ] 显示选项里有一个总开关式的极简模式。
- [ ] 极简模式是单独的一块，不和下面那组细节显示开关混排。
- [ ] 极简模式打开时，下方细节显示开关不再生效，树回到更克制的基础显示。
- [ ] 极简模式打开时，树里的值仍然显示。
- [ ] 极简模式关闭时，下方细节显示开关重新按各自配置生效。
- [ ] 链接按钮有单独开关，默认打开。
- [ ] 链接按钮属于值旁边的附加显示，会受极简模式影响。
- [ ] iframe 模式仍然保留显示选项，不允许直接砍掉。
- [ ] 显示选项至少包含：图标、数组长度、数组索引、值预览、图片预览、文件夹图标。
- [ ] 图片预览开关会真实影响树上的缩略预览显示。

### G. 模式隔离

- [ ] embedded iframe 模式有自己清晰的控件集合，不继承杂乱的 standalone 控件。
- [ ] detached viewer 的工作不会无意改坏 iframe 行为。
- [ ] raw/pretty 的工作不会无意改坏 iframe 行为。

### H. Raw / Pretty 新窗口处理

- [ ] 像 `raw`、`pretty` 这类值本身可解析成 JSON 的字符串，在树值旁边有明显的打开入口。
- [ ] 多行 `pretty` 值也可以直接在新窗口 viewer 里打开。
- [ ] 打开时通过扩展页 URL 参数把原始 JSON 文本传进 detached viewer。
- [ ] 打开时会同时带上来源 path 和来源页面 URL，便于多标签页区分来源。
- [ ] 新窗口继续使用 iframe takeover 那套界面壳，而不是 standalone viewer 的另一套布局。
- [ ] 打开后的 detached viewer 会把该字符串当成一份新的 JSON 文档，而不是普通字符串。
- [ ] 只有首字符明确是 `[` 或 `{` 的字符串值，才允许进入下一层。
- [ ] 像 `12.34` 这类普通数字字符串，不算下一层，也不应出现该入口。
- [ ] detached viewer 的标签标题至少包含来源 path、来源 URL 和 `JSON Mate Viewer`。

- [ ] E2E：iframe 接管能正常加载
- [ ] E2E：点击树行会同步更新 `Path`、`Key` 和编辑器值
- [ ] E2E：面板收起和重新展开可用
- [ ] E2E：右上角 logo handle 会收起整个右侧工作区背景
- [ ] E2E：搜索能打开、聚焦、持续输入，并能选择结果
- [ ] E2E：URL 字段会显示行内跳转入口
- [ ] E2E：图片字段显示预览，而不是被当普通 URL
- [ ] E2E：`Expand all / Collapse all` 会真实改变树内容展开状态
- [ ] E2E：极简模式会接管下方显示选项
- [ ] E2E：iframe 显示选项仍存在，且图片预览开关生效
- [ ] E2E：手工编辑后 `undo/redo` 可往返恢复
- [ ] E2E：`Single line` 会明显改变编辑器表现

## 当前事实快照

下面这些是当前已知问题，不能再含糊带过：

- [ ] embedded viewer 仍有部分非 embedded 控件和结构泄漏。
- [ ] 某些 URL 入口之前放错了位置，不在用户第一眼会看的区域。
- [ ] 有些交互虽然“代码里存在”或“测试里存在”，但真实页面里并不明显。
- [ ] 搜索行为已经不止一次被 rerender / focus 时序回归搞坏。
- [ ] 树行命中区域仍需要更严格的真实浏览器验证。

## 交付流程

iframe 相关工作按下面的循环推进：

1. 只挑一个未完成项，或一组强相关的小项。
2. 只实现这一个切片。
3. 在真实浏览器里验证。
4. 增加或更新自动化覆盖。
5. 只有上面两者都成立，才标记为完成。

超出这份清单的内容，一律视为独立 scope。
